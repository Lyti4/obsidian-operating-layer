# Runbook — `obsidian_backfill_report.py`

## Назначение

Инструмент записывает итоговый Markdown-отчёт в явно выбранный каталог Obsidian
Reports. Это approved-write (утверждённая запись): сам факт успешного proposal
или apply не даёт разрешения создавать или перезаписывать отчёт.

## До запуска

1. Получить явное разрешение на точный output-файл.
2. Проверить, что proposal относится к принятому срезу и не содержит секретов.
3. Указать `--out` явно; не полагаться на общее имя по умолчанию.
   Инструмент технически откажет, если resolved path выходит за
   `--reports-dir`.
4. Выбрать новое уникальное имя: инструмент откажется перезаписывать
   существующий файл.
5. Записать факт отсутствия output-файла как исходную точку отката.

## Запуск

```bash
python3 tools/obsidian_backfill_report.py \
  --proposal /absolute/path/to/proposal.json \
  --reports-dir /absolute/path/to/Obsidian/Reports \
  --out /absolute/path/to/Obsidian/Reports/approved-report.md
```

## Проверка

- Выведенный путь совпадает с одобренным `--out`.
- Изменён ровно один отчётный файл.
- В отчёте нет token, credential, private URL или полного текста заметок.
- Ссылки на proposal, backup и проверку относятся к текущему срезу.

## Откат

- Удалить только созданный этим запуском одобренный output-файл.
- После отката убедиться, что другие заметки и `.obsidian` не изменились.

## Стоп-условия

Нет явного approval, `--out` не внутри resolved `--reports-dir`, output уже
существует, proposal устарел, либо отчёт содержит чувствительные данные.
