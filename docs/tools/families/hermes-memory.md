# Семейство `hermes-memory`

## Зачем

Это read-only (только чтение) поверхность, через которую Hermes сможет получать
короткий контекст из Obsidian. Сейчас она содержит только лексический fallback
(резервный поиск по словам), а не активный MemoryProvider (встроенный провайдер
памяти Hermes).

## Кто и что входит

- Hermes формулирует запрос и решает, использовать ли найденный контекст.
- `tools/hermes_obslayer_memory.py` ищет Markdown-заметки и возвращает не более
  заданного лимита результатов.
- Владелец live vault и разрешений остаётся человек; поиск не даёт права записи.

## Границы данных

- Читаются только `*.md` внутри явно выбранного vault.
- Исключаются `.obsidian`, `.trash`, `_Backups`, `_Archive` и `__pycache__`.
- Файлы больше 2 MB и symlink (символическая ссылка), выходящий из vault,
  пропускаются.
- В ответе допустимы относительный путь, заголовок, score и короткий фрагмент.
- Запрещено сохранять полный текст заметок, секреты или приватные URL в логах.

## Текущий вызов

```bash
python3 tools/hermes_obslayer_memory.py "текст запроса" --vault /path/to/vault --limit 8 --json
python3 tools/hermes_obslayer_memory.py --self-test
```

## Будущий native provider

Подключение к Hermes выполняется отдельным срезом. Provider обязан:

1. вызывать только эту read-only поверхность или её проверенного преемника;
2. иметь небольшой лимит результатов и символов контекста;
3. иметь bounded timeout (ограниченное время ожидания);
4. работать fail-open (при ошибке вернуть пустой контекст и не блокировать
   основной ответ Hermes);
5. не индексировать и не отправлять наружу заметки самовольно;
6. иметь отдельный тест конфигурации до включения сервиса.

## Проверка и отказ

- Тест: `tests/test_hermes_obslayer_memory.py`.
- Спецификация: `docs/spec-kit/51-hermes-native-memory-first-slice.md`.
- Остановиться при выходе пути за vault, пустом запросе, недоступном vault,
  превышении лимита или обнаружении secret-shaped (похожих на секрет) данных.
- Семантика, автоматический intake и голос сюда пока не входят; их нельзя
  объявлять работающими по наличию лексического поиска.
