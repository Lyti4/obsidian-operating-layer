# Инструкции для safety-core `src/obslayer/`

## Область действия

Этот каталог содержит переиспользуемые контракты и guardrails (защитные
ограничители). Его правила наследуют корневой `AGENTS.md` и применяются ко всем
вызывающим CLI.

## Неизменяемые границы

- После нормализации путь остаётся внутри разрешённого корня.
- Dry-run (пробный запуск без записи) сохраняется режимом по умолчанию для
  потенциально изменяющих операций.
- Живая запись разрешена только через утверждённый apply-контур с точной
  привязкой proposal, manifest, vault и targets.
- Защищённые пути и секретные данные fail closed (отказывают безопасно).
- Ошибки сообщают путь/тип проверки, но не раскрывают защищённое содержимое.

## Изменение контракта

Сначала добавьте воспроизводящий или негативный тест, затем минимальное
исправление. В том же срезе обновите вызывающие инструменты,
`docs/TOOLS_POLICY.md`, `docs/tools/INDEX.md`, семейное руководство и связанный
runbook. Запустите узкий тест и `make verify`.
