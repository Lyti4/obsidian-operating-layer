# Инструкции для `tools/`

## Область действия

Этот файл применяется к каждому `tools/*.py` и наследует корневой
`AGENTS.md`. Инструмент не получает разрешение только потому, что код технически
может выполнить действие.

## Контракт инструмента

- `cli` (команда) имеет понятный `--help`, стабильный код возврата и безопасное
  значение по умолчанию.
- `internal` (внутренний модуль) не документируется как пользовательская
  команда и покрывается через вызывающие тесты.
- Read-only (только чтение) не создаёт побочных файлов.
- Proposal-only (только предложение) пишет лишь в явно выбранный отчётный или
  proposal-каталог.
- Sandbox (песочница) работает только с копией или производными данными.
- Approved-write (утверждённая запись) требует отдельного runbook, явного
  разрешения, резервной копии и проверки.

## Безопасность

- Не выходите за разрешённые корни после `resolve()`.
- Не печатайте секреты, приватные URL или тела заметок в диагностике.
- Live vault не является тестовой фикстурой.
- Запись в живой vault проходит только через утверждённый apply-контур.
- Сетевой вызов, сервис, cron, auth или платный API требуют отдельной границы и
  согласования.

## Обязательное сопровождение

Любое добавление, удаление, переименование или изменение контракта инструмента
в том же срезе обновляет:

- строку в `docs/tools/INDEX.md`;
- одно основное семейство под `docs/tools/families/`;
- тест или явную ссылку `covered-by:` для внутреннего модуля;
- подробную спецификацию или runbook, если они требуются;
- итоговый отчёт о влиянии на документацию.

Новый `tools/*.py` без строки реестра считается незавершённым изменением.
